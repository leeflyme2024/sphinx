# memtester
```bash

Usage: memtester [-p physaddrbase [-d device]] <内存大小>[B|K|M|G] [循环次数]
```

`memtester` 是一个用于检测物理内存稳定性和可靠性的工具，通过反复读写内存来发现潜在的硬件问题。以下是其用法详解：

### **基本语法**
```bash
memtester [-p physaddrbase [-d device]] <内存大小>[B|K|M|G] [循环次数]
```

### **参数详解**

1. **`-p physaddrbase`**  
   - 指定要测试的物理内存起始地址（十六进制）。  
   - 通常用于嵌入式系统或直接测试特定物理内存区域（如硬件内存模块）。  
   - **示例**：`-p 0x80000000` 表示从物理地址 `0x80000000` 开始测试。

2. **`-d device`**  
   - 指定访问物理内存的设备文件（如 `/dev/mem`）。  
   - 需与 `-p` 参数配合使用，且需要 root 权限。  
   - **示例**：`-d /dev/mem`。

3. **`<内存大小>`**  
   - 要测试的内存量，支持单位：  
     - `B`：字节（Byte）  
     - `K`：千字节（KB）  
     - `M`：兆字节（MB）  
     - `G`：千兆字节（GB）  
   - **示例**：`100M` 表示测试 100MB 内存，`2G` 表示测试 2GB 内存。

4. **`[循环次数]`**  
   - 测试循环次数（默认为 1 次）。  
   - 若不指定，测试完成后自动退出。  
   - **示例**：`5` 表示循环测试 5 次。

### **常用场景示例**

1. **基础测试（用户空间内存）**  
   测试 1GB 内存，循环 3 次：
```bash
memtester 1G 3
```

2. **直接测试物理内存（嵌入式系统）**  
   从物理地址 `0x80000000` 开始测试 512MB 内存：
```bash
memtester -p 0x80000000 -d /dev/mem 512M
```

3. **最大内存压力测试**  
   测试系统所有可用内存（需确保系统有足够空闲内存）：
```bash
memtester all
```

### 通用控制选项
#### 基本控制
- `--abort`: 如果任何一个压力测试失败，则终止所有压力测试。
- `--aggressive`: 启用所有激进选项，以增加系统负载。
- `-a N, --all N`: 对每个压力测试启动N个工作线程。
- `-b N, --backoff N`: 在工作开始前等待N微秒。
- `--class name`: 指定一类压力测试，与`--sequential`一起使用。
- `-n, --dry-run`: 不实际运行，仅显示将要执行的压力测试。
- `--ftrace`: 启用内核函数调用跟踪。
- `-h, --help`: 显示帮助信息。
- `--ignite-cpu`: 修改内核控制使CPU过热运行。
- `--ionice-class C`: 指定io优先级类（idle、besteffort、realtime）。
- `--ionice-level L`: 指定io优先级级别（0最大，7最小）。
- `-j, --job jobfile`: 运行指定的作业文件。
- `-k, --keep-name`: 保持压力测试工作线程名称为'stress-ng'。

#### 日志和输出
- `--log-brief`: 减少日志消息的冗长度。
- `--log-file filename`: 将日志消息记录到文件。
- `-q, --quiet`: 安静模式，减少输出。
- `--verbose, -v`: 详细输出。
- `--timestamp`: 时间戳记日志输出。
- `--times`: 在运行结束时显示运行时间摘要。
- `--syslog`: 将日志消息记录到系统日志。
- `--verify`: 验证结果（并非所有测试都可用）。
- `-V, --version`: 显示版本信息。
- `--yaml file, -Y`: 将结果输出到YAML格式的文件。

#### 调度和资源管理
- `--maximize`: 启用最大限度的压力选项。
- `--minimize`: 启用最低限度的压力选项。
- `--no-madvise`: 不对每个mmap使用随机madvise选项。
- `--oomable`: 如果某个压力测试被OOM杀死，不重新启动它。
- `--page-in`: 触摸不在核心中的已分配页面。
- `--parallel N`: 同义词为'all N'。
- `--pathological`: 启用可能导致机器挂起的压力测试。
- `--perf`: 显示性能统计信息。
- `--sched type`: 设置调度类型。
- `--sched-prio N`: 设置调度优先级级别N。
- `--sched-period N`: 设置SCHED_DEADLINE的周期为N纳秒（仅限Linux）。
- `--sched-runtime N`: 设置SCHED_DEADLINE的运行时间为N纳秒（仅限Linux）。
- `--sched-deadline N`: 设置SCHED_DEADLINE的截止时间为N纳秒（仅限Linux）。
- `--sched-reclaim`: 为deadline调度器设置回收CPU带宽（仅限Linux）。
- `--taskset`: 使用特定的CPU（设置CPU亲和性）。
- `--temp-path path`: 指定临时目录和文件的路径。
- `--thrash`: 强制加载所有页面导致交换颠簸。
- `-t N, --timeout T`: 在T秒后超时。
- `--timer-slack`: 启用计时器松弛模式。

- `--vm-populate`: 预分配（prefault）页表以映射。

#### 地址空间练习
- `--vm-addr N`: 启动N个工作线程，练习虚拟内存地址。
- `--vm-addr-ops N`: 执行N次虚拟内存地址操作后停止。

#### 进程间内存复制
- `--vm-rw N`: 启动N个工作线程，使用process_vm* API进行读写操作。
- `--vm-rw-bytes N`: 每次bogo操作传输N字节的内存。
- `--vm-rw-ops N`: 执行N次process_vm*复制操作后停止。

#### 地址空间解除映射
- `--vm-segv N`: 启动N个工作线程，解除其地址空间的映射。
- `--vm-segv-ops N`: 解除映射后的N次SEGV故障后停止。

#### 使用vmsplice读写
- `--vm-splice N`: 启动N个工作线程，使用vmsplice进行读写操作。
- `--vm-splice-ops N`: 执行N次splice操作后停止。
- `--vm-splice-bytes N`: 每次vmsplice调用传输的字节数。

#### 等待子进程停止/恢复
- `--wait N`: 启动N个工作线程，等待子进程停止或恢复。
- `--wait-ops N`: 执行N次等待操作后停止。

#### 看门狗设备
- `--watchdog N`: 启动N个工作线程，练习/dev/watchdog设备。
- `--watchdog-ops N`: 执行N次看门狗操作后停止。

#### 宽字符字符串函数
- `--wcs N`: 启动N个工作线程，练习C库宽字符字符串函数。
- `--wcs-method func`: 指定要练习的宽字符字符串函数。
- `--wcs-ops N`: 执行N次宽字符字符串操作后停止。

#### 使用syscall函数
- `--x86syscall N`: 启动N个工作线程，使用syscall函数。
- `--x86syscall-ops N`: 执行N次syscall函数调用后停止。
- `--x86syscall-func F`: 仅使用syscall函数F。

#### 文件扩展属性
- `--xattr N`: 启动N个工作线程，练习文件扩展属性。
- `--xattr-ops N`: 执行N次扩展属性操作后停止。

#### 调度器yield()调用
- `-y N, --yield N`: 启动N个工作线程，执行sched_yield()调用。
- `--yield-ops N`: 执行N次yield操作后停止。

#### /dev/zero读取
- `--zero N`: 启动N个工作线程，读取/dev/zero。
- `--zero-ops N`: 执行N次/dev/zero读取操作后停止。

#### 数据压缩
- `--zlib N`: 启动N个工作线程，使用zlib压缩数据。
- `--zlib-level L`: 指定zlib压缩级别0=快速, 9=最佳。
- `--zlib-mem-level L`: 指定zlib压缩状态内存使用量1=最小, 9=最大。
- `--zlib-method M`: 指定zlib随机数据生成方法M。
- `--zlib-ops N`: 执行N次zlib压缩操作后停止。
- `--zlib-strategy S`: 指定zlib策略0=default, 1=filtered, 2=huffman only, 3=rle, 4=fixed。
- `--zlib-stream-bytes S`: 指定当前流关闭前需要解压的字节数。
- `--zlib-window-bits W`: 指定zlib窗口位数 -8-(-15) | 8-15 | 24-31 | 40-47。

#### 僵尸进程
- `--zombie N`: 启动N个工作线程，快速创建和回收僵尸进程。
- `--zombie-ops N`: 创建N次僵尸进程fork操作后停止。
- `--zombie-max N`: 设置每个工作线程的僵尸进程上限为N。
